Back to the future !

JavaScript est un langage de programmation interprété créé en 1995 par Netscape. Il a été conçu pour ajouter de l'interactivité aux pages Web, et est rapidement devenu l'un des langages de programmation les plus populaires.

Fin des années 1990, début des années 2000, JavaScript a été principalement utilisé pour des tâches simples comme par exemple la validation des formulaires. Cependant, avec la montée en puissance des navigateurs web, JavaScript a su gérer des tâches plus complexes, comme la création d'animations et de jeux.

Au milieu des années 2000, l'émergence d'applications Web a conduit au développement de librairies et de frameworks tels que jQuery et AngularJS, ce qui a permit le développement d' applications plus complexes.

Aujourd'hui, JavaScript est un élément clé du web moderne. Il est tilisé pour créer des sites interactifs et dynamiques, construire des applications mobiles et créer des applications côté serveur (grâce à Node JS). Il peut également être utilisé en conjonction avec d'autres technologies telles que WebGL pour créer des graphismes 3D pour les applications web.

Les standards du langage qui constituent la norme ECMAScript sont définis par le groupe ECMA International

Les bases

Javascript est un langage interprété qui s'exécute (principalement) côté client, c'est à dire dans le navigateur. Principalement, parce que depuis la création en 2009 de Node.js par Ryan Dahl, il peut aussi être exécuté côté serveur.

Où écrire le code Javascript ?

  • directement à l'intérieur de balises <script></script>
  • dans un fichier externe

Les balises <script></script>, peuvent être placées entre les balises <head></head> ou directement dans le <body></body>

Par exemple :


<head>
  <script>
    alert("Hello From the Head");
  </script>
</head>
<body>
  <script>
    alert("Hello From the Body");
  </script>
</body>

Dans le cas d'un fichier externe, on utilise une balise <script></script> dont l'attribut src contient le chemin vers le fichier.

Une bonne pratique est de positionner cette balise dans le <head></head> avec un attribut defer qui permet de retarder l'exécution du script jusqu'à ce que l'analyse du document soit terminée. 


<head>
  <script src="js/app.js" defer></script>
</head>

2 avantages à ça :

  • pas de blocage du parsing du HTML par le navigateur
  • lorsque le script est exécuté, la structure HTML a déjà été analysée

On peut aussi écrire et exécuter du code Javascript directement dans la console du navigateur (outils de dévelopement)

Pour des raisons de performance et de lisibilité, il est préférable d'externaliser les scripts. la séparation des différents langages ou codes est aujourd’hui la norme pour essayer de conserver un ensemble le plus propre, le plus compréhensible et le plus facilement maintenable possible.

Programming building blocks

Variables

Une variable c'est en gros un conteneur nommé qui permet de stocker des données en mémoire.

On utilise les mots-clés suivants pour déclarer des variables:

  • var
  • let
  • const

var et let ont le même objectif mais présentent quelques différences, dont la principale est leur portée.

La portée d'une variable déclarée avec var est locale au contexte dans lequel elle a été déclarée. Par exemple, si elle est déclarée dans une fonction, sa portée sera locale a celle-ci et éventuellement aux fonctions qui seraient imbriquées. Si elle est déclarée en dehors d'une fonction, sa portée sera dite globale.

Les variables déclarées avec let par contre, ont une portée bloc, ce qui signifie qu'elles ne sont visibles que dans le bloc dans lequel elles sont déclarées. Elle seront également globales si elles sont déclarées en dehors d'un bloc.

Voici un exemple qui illustre la différence entre var et let : 


function foo() {
    if (true) {
    var x = 5;
    console.log(x); // prints 5
    }
    console.log(x); // prints 5
}

function bar() {
    if (true) {
    let y = 5;
    console.log(y); // prints 5
    }
    console.log(y); // ReferenceError: y is not defined
}

Dans la fonction foo, la variable var x est déclarée à l'intérieur d'un bloc, mais est toujours visible à l'extérieur de ce bloc.

Dans la fonction bar par contre, la variable let y est déclarée dans le bloc if, et elle n'est pas visible à l'extérieur du bloc.

Une autre différence entre var et let est que les variables var sont sujettes au hoisting (remontée), ce qui signifie qu'elles peuvent être consultées avant qu'elles ne soient déclarées (elles auront alors pour valeur undefined).

Les variables let, par contre, ne sont pas hoisted, et ne peuvent pas être consultées avant leur déclaration.

Par exemple :


console.log(x); // undefined
var x = 5;

console.log(y); // ReferenceError: y is not defined
let y = 5;

Dans le premier exemple, la variable var x est accessible avant qu'elle ne soit déclarée, mais elle n'est pas initialisée. C'est parce que les variables var sont automatiquement initialisées à undefined lorsqu'elles sont déclarées.

Dans le deuxième exemple, on accède à la variable let y avant qu'elle ne soit déclarée et une erreur de type ReferenceError est levée.

Une autres différence entre var et let est le fait que les variables let ne peuvent pas être rédéclarées dans le même bloc (une erreur sera levée), alors que les variables var peuvent l'être.

Prenez l'habitude de déclarer vos variables avec let, les règles de portée vous éviteront les re-déclarations accidentelles et d'autres types de bugs.

Notez quand même que var est toujours largement utilisé et que vous pouvez le rencontrer par exemple dans du code legacy.

const est le mot clé utilisé pour déclarer des constantes en JavaScript.

Les constantes ont également une portée bloc, comme les variables let, mais on ne peut pas leur réaffecter de valeur après leur déclaration.

En clair, une fois que vous attribuez une valeur à une constante, vous ne pouvez pas la modifier.

Un exemple:


const PI = 3.14;
PI = 3.14159; // TypeError: Assignment to constant variable.

// If you try to reassign a value to a const variable, a TypeError will be thrown.

Une chose importante à noter sur les constantes : elles ne sont pas immuables. Cela signifie que si vous assignez un objet ou un tableau à une constante, vous pouvez toujours modifier les modifier.

Par exemple:


const arr = [1, 2, 3];
arr.push(4); // [1, 2, 3, 4]

Ici on affecte un tableau à la constante arr, et on est en mesure de modifier le tableau en utilisant la méthode push. Par contre, on ne peut pas réaffecter un nouveau tableau ou une valeur différente à arr.

En résumé, utilisez const pour déclarer des variables dont vous ne voulez pas modifier la valeur, et utilisez let quand vous devez réaffecter la valeur.

Primitives

Une primitive est une donnée qui n'est pas un objet et n'a pas de méthode.

En JavaScript, les valeurs primitives sont non mutables, c'est à dire qu'elles ne peuvent pas être modifiées une fois qu'elles sont créées, au contraire des objets et tableaux, qui sont eux mutables et peuvent être modifiés après leur création.

Il existe 7 types de primitives :

  • string
  • number
  • bigint
  • boolean
  • undefined
  • symbol
  • null

Essayer de modifier une valeur primitive n'a aucun effet, parce que la valeur elle-même ne peut pas être changée. On peut par contre créer une nouvelle valeur primitive avec une valeur différente.

Pour aller plus loin à propos de la notion de mutabilité : mutability-and-immutability-in-javascript-explained-in-detail

data types

En JavaScript, il existe plusieurs types de données que vous pouvez utiliser pour stocker des valeurs.

Voici une liste des différents types de données :

  • Number: Un type de données Number représente des valeurs numériques. En JavaScript, il n'y a qu'un seul type de nombre, et il peut être soit un entier, soitun décimal (float).
  • String: Un type de données String représente une chaîne de caractères. Une chaîne de caractère peut être définie à l'aide de guillemets simples (') ou doubles ("). On peut aussi utiliser la notation (`) ou Template Literal qui permet d'interpoler des variables.
  • Boolean: Un type de données booléen représente une valeur vraie ou fausse.
  • Null: Le type de données null représente une absence intentionnelle de valeur.
  • Undefined: Le type de données undefined représente une valeur non initialisée.
  • Object: Un type de données Object représente une collection de données et de fonctions qui sont liées.
  • Symbol: Un type de données de symbole est un identifiant unique et immuable.

En plus de ces types de base, JavaScript dispose également de quelques types de données particuliers, comme fonction et array, qui sont en fait des objets ayant des caractéristiques et des fonctionnalités supplémentaires.

Il est important de comprendre les différents types de données en JavaScript, car différents types de données nécessitent différentes opérations et méthodes.

Array

En JavaScript, un tableau est un type de données qui représente une liste ordonnée de valeurs de tous types.

Voici un exemple de tableau :


const numbers = [1, 2, 3, 4, 5];

Dans cet exemple, le tableau de nombres comporte cinq éléments : 1, 2, 3, 4 et 5.

Les éléments d'un tableau sont numérotés, à partir de 0, et vous pouvez y accéder à l'aide de la notation crochet : 


console.log(numbers[0]); // prints 1
console.log(numbers[1]); // prints 2
console.log(numbers[2]); // prints 3

Vous pouvez aussi modifier les éléments d'un tableau :


numbers[2] = 100; // modify an element
numbers[5] = 6; // add a new element
numbers.length; // prints 6

Vous pouvez utiliser des tableaux pour stocker des listes de données (comme une liste de noms ou une liste de nombres), ou vous pouvez les utiliser pour stocker des structures de données plus complexes (comme une liste d'objets).

Voici un exemple d'un tableau contenant des valeurs mixtes:


const mixed = [1, "string", true, [2, 3], { key: "value" }];

Dans cet exemple, le tableau mixed contient 5 éléments: un number, une string, un boolean, un array, et un object. On peut accéder aux éléments du tableau en utilisant la notation crochet : 


console.log(mixed[0]); // prints 1
console.log(mixed[1]); // prints 'string'
console.log(mixed[2]); // prints true
console.log(mixed[3]); // prints [2, 3]
console.log(mixed[4]); // prints {key: 'value'}

On peut également accéder aux éléments d'un tableau imbriqué et d'un objet en utilisant la même notation combinée à la notation point : 


console.log(mixed[3][0]); // prints 2
console.log(mixed[4].key); // prints 'value'

Les tableaux peuvent contenir des éléments de tout type, y compris des primitives, des tableaux et des objets. Cela en fait un type de données puissant et flexible car ils permettent de stocker et de manipuler une large gamme de données en un seul endroit.

Object

En JavaScript, un objet est un type de données qui est une entité contenant un ensemble de variables (propriétés) et de fonctions (méthodes). Les objets peuvent interagir entre eux. Un objet peut être considéré comme un conteneur pour les propriétés, où chaque propriété a un nom et une valeur.

La valeur d'une propriété peut être un type de données primitif (comme un nombre ou une chaîne de caractères), ou il peut être un type de données plus complexe (comme une fonction ou un autre objet).

Un exemple d'objet ici :


const person = {
  name: "John",
  age: 30,
  greet: function () {
    console.log("Hello, my name is " + this.name);
  },
};

Dans cet exemple, l ' objet a trois propriétés: name , age et greet . Les propriétés name et age contiennent des valeurs primitives (une chaîne de caractères et un nombre, respectivement), et la propriété greet quand à elle contient une fonction.

Vous pouvez accéder aux propriétés d'un objet en utilisant la notation point ou la notation crochet : 


console.log(person.name); // prints 'John'
console.log(person["age"]); // prints 30

On peut aussi ajouter, modifier ou supprimer des propriétés d'un objet: 


person.location = "New York"; // add a new property 
person.age = 31; // modify an existing property 
delete person.greet; // delete a property

Comme les tableaux, les objets sont un type de données puissant et qui vous permettent de regrouper des données et des fonctions liées et de les traiter en tant qu'entité unique.

Vous pouvez utiliser des objets pour représenter des objets du monde réel (comme une personne ou une voiture), ou des structures personnalisées pour stocker et manipuler des données dans vos scripts.

Opérateurs

Arithmétiques

  • + : Addition
  • - : Soustraction
  • * : Multiplication
  • / : Division
  • % : Modulo (reste de la division)
  • ++ : Incrément
  • -- : Décrément

Il existe deux façons d'utiliser l'opérateur ++ : pré-incrément et post-incrément.

L'opérateur pré-incrément (++x) augmente la valeur de x puis retourne la nouvelle valeur. Par exemple : 


let x = 5;
console.log(++x); // prints 6
console.log(x); // prints 6

L'opérateur post-incrément (x++) retourne la valeur courante de x, puis augmente la valeur de x. 


  let x = 5;
  console.log(x++); // prints 5
  console.log(x); // prints 6

Même chose pour l'opérateur --

Il est important de comprendre la différence entre le pré-incrément et le post-incrément, car l'ordre dans lequel la valeur est incrémentée et retournée peut affecter le comportement de votre script.

Affectation

  • = : Affectation simple
  • += : Addition et affectation
  • -= : Soustraction et affectation
  • *= : Multiplication et affectation
  • /= : Division et affectation
  • %= : Modulo et affectation

Comparaison

  • == : Egalité (valeur)
  • != : Inégalité
  • === : Egalité stricte (valeur et type)
  • !== : Inégalité stricte
  • > : Strictement supérieur
  • < : Strictement inférieur
  • >= : Supérieur ou égal
  • <= : Inférieur ou égal

Logique

  • && : Et
  • || : Ou
  • ?? : Opérateur de coalescence des nuls

Divers

  • typeof : Retourne le type d'une valeur
  • instanceof : Renvoie vrai si un objet est une instance d'un constructeur spécifique

Et il en existe encore bien d'autres. Plus approfondir le sujet : Liste des opérateurs Javascript (MDN)

Les différents types d'opérateurs vous permettent d'effectuer diverses opérations sur les valeurs et de contrôler le flux de votre code.

Expressions conditionnelles

Vous pouvez utiliser des expressions conditionnelles pour exécuter différents blocs de code basés sur le fait qu'une condition est évaluée comme vraie ou fausse.

Voici une liste des principales structures conditionnelles en JavaScript :

  • if : permet d'exécuter un bloc de code si une certaine condition est vraie. 
    
if (condition) {
  // code to be executed if condition is true
}
  • if...else : L'instruction if...else permet d'exécuter un bloc de code si une certaine condition est vraie, et un bloc différent si la condition est fausse. 
    
if (condition) {
  // code to be executed if condition is true
} else {
  // code to be executed if condition is false
}
  • ? ... : : l'opérateur ternaire est utilisé comme un raccourci de la structure if...else 
    
let age = 20;
let isAdult = (age > 18) ? true : false;
console.log(isAdult) // will output true since age > 18
  • if...else if...else : permet de spécifier plusieurs conditions et d'exécuter différents blocs de code selon quelle condition est vraie. 
    
if (condition1) {
  // code to be executed if condition1 is true
} else if (condition2) {
  // code to be executed if condition1 is false and condition2 is true
} else {
  // code to be executed if condition1 and condition2 are false
}
  • switch : switch vous permet de spécifier plusieurs conditions et d'exécuter différents blocs de code selon la valeur d'une certaine expression. 
    
switch (expression) {
  case value1:
    // code to be executed if expression === value1
    break;
  case value2:
    // code to be executed if expression === value2
    break;
  default:
  // code to be executed if expression does not match any case
}

Boucles

Les boucles sont utilisées pour exécuter un bloc de code à plusieurs reprises.

  • for : La boucle for permet de spécifier un compteur de boucle et d'exécuter un bloc de code pour un nombre spécifié d'itérations. 
    
for (let i = 1; i <= 10; i++) {
  // code to be executed as long as i is <= 10
}
  • forEach : La méthode forEach permet d'exécuter une fonction pour chaque élément dans un tableau. 
    
array.forEach(function (element) {
  // code to be executed for each element
});
